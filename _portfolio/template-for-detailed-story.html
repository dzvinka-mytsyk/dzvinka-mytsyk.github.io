Rarely development teams are working on User Stories containing only in 3 sentences: “As a... I want... So that...”

In most cases User Stories need mock-ups, Acceptance Criteria, design documents, problem statements and/or other details to be discovered and documented.

This detailed information can be researched and documented before/after/as a result of Backlog Grooming meeting, and the document below contains the template of such detailed User Story structure.
